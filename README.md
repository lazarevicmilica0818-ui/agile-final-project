## GitHub Project (Kanban Board)
The Kanban board includes the following workflow columns:
- Backlog
- Ready
- To Do
- In Progress
- In Review
- Done

The board was used to simulate Agile workflow by moving user stories through the development lifecycle.

---

## Product Backlog (User Stories)
- User Registration
- User Login
- View Menu Items
- Add to Cart
- Place Order

---

## Sprint Planning (Sprint 1)
Sprint 1 focused on basic user functionality.

Selected stories:
- User Registration (3 points)
- User Login (2 points)
- View Menu Items (3 points)

Total Sprint Velocity: 8 story points

Sprint Goal: Implement basic user authentication and menu browsing functionality.

---

## Sprint Execution
Tasks were moved across the Kanban board following Agile workflow:
To Do → In Progress → In Review → Done

---

## Sprint Review
- Completed: User Registration, User Login, View Menu Items  
- Total Story Points: 8  
- Sprint Goal Achieved: Basic user functionality implemented

## Project Overview
This project demonstrates Agile methodology using GitHub Projects. It includes product backlog management, sprint planning, Kanban board execution, and sprint review & retrospective.

The application simulated is a simple food ordering system.
